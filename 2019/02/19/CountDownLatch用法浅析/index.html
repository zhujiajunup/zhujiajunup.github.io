CountDownLatch用法浅析

发表于 2019-02-19 | 分类于 java ， 多线程 | 热度 ℃

前言

昨天线上服务出现了一个问题，使用spring boot的数据同步每日周期任务，只用重启后的一两天会被调度，之后就没有调度信息了。
查看代码发现，数据同步代码中使用了CountDownLatch，初始值为待同步数据的记录数n，而在同步过程中根据指定条件
过滤了一些记录，这就导致了实际同步的记录数m可能小于待同步的记录数n，而在最后使用了CountDownLatch.await()操作来等待同步完成，这就导致了在m 比n的情况下，await永远不会退出，线程就一直在等待。
在这样的情况发生后，之后的周期任务就不会被调度了。

CountDownLatch

CountDownLatch在java.util.concurrent包下，允许一个或多个线程等待直到在其他线程执行的操作完成后再进行后续操作的工具类。
CountDownLatch类非常简单

CountDownLatch在初始化的时候需要指定一个大于0的计数n，n代表线程数

每个线程调用CountDownLatch.countDown()，该计数会-1

应用场景

  • 在等待某些(一个或多个)条件满足后（如MailServer启动后）才能进行后续的操作（发邮件）
